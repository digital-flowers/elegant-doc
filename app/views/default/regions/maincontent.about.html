Why Elegant?

Everyone Knows that NodeJS Is Taking The Lead! taking Javascript to the server side development with its v8 javascript engine it is taking over the Enterprise. For the past two years NodeJS had huge growth and support from the open source community, currently in use by Microsoft, VMWare, Ebay, Yahoo, and many more top tech companies.

Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient, but this also make it very difficult to write and learn, and may lead the developers to write an ugly and hard to understand code specially what is known as callback hell,for example checkout these questions from the stackoverflow that most nodejs developers are facing:

- Callback hell in nodejs?
- How to avoid long nesting of asynchronous functions in Node.js
- Event Driven programming - non linear code

This is why elegant framework was built with a totally new concept that makes NodeJS code more friendly and easier for building web applications without losing the efficiency of event-driven and non-blocking I/O model.

Flexible Framework

The core of elegant framework is build in a way that makes it easy to customize without any need from the developers to miss up with the core system. this core is built in a new concept called the interceptors stack, every request will pass through this stack , and this stack will handle the page life cycle , each interceptor is a block of code that handles a certain job and has its own weight to determinate when it will be executed.

The elegant interceptors stack is totally flexible , the Developer will have the ability to override any core interceptor or add a new one , that's why elegant framework is very flexible.
